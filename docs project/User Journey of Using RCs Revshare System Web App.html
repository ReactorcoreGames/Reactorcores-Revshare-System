User Guide: Using the Revshare System Web App

This guide walks you through how to use the Revenue Share System web application as a project leader. The app is designed to help you manage your team, calculate fair payouts, and maintain complete transparency throughout your project's lifetime.

Overview

The Revshare System web app is a standalone tool that works entirely offline. All your data is stored in a team-project data file that you keep on your local device. This file contains:

  • All team member information and tier assignments
  • Complete payout history and timeline
  • Project introduction and settings

You can share read-only copies of this file with your team for transparency, and you can load it back into the app anytime to continue managing your project.

Only the file managed by the project leader is the valid one, known as the "master file". To prevent anyone from tampering with the team-project data file, the project leader must always keep their version of it secure and backed up without anyone else allowed to change it. Any copies of the file provided by other members should generally be considered untrustworthy and rejected unless its an emergency.

Step-by-Step User Journey

1First Time Setup

When you first open the web app:

  1. Read the introduction to understand what the system does and how it works
  2. Review the Quick Start guide to get the gist of when to use it and how to use it
  3. Fill out all details for your team:
    • Team member names, roles, and contact information
    • Tier assignments (Main, Assistant, Special Thanks, Fan)
    • Payment preferences for each member
  4. Add your project information:
    • Project name and description
    • Any custom settings or notes
  5. Save your data to create your first team-project data file

3Loading Your Data (Returning Sessions)

When you open the web app again after closing it:

The app won't have any data loaded by default (this is intentional for privacy and security). To continue where you left off:

Option A: Drag & Drop

  • Simply drag your team-project data file from anywhere on your computer
  • Drop it anywhere on the web app interface
  • The app will detect and load your data instantly

Option B: Browse & Select

  • Click the "Load Data" or "Import File" button
  • Browse your computer to find your team-project data file
  • Select it to load all your previous data

Why this system? The web app works standalone and offline, with no cloud storage or cache. This gives you complete control over your data and allows you to easily share read-only copies with team members for transparency.

4Ongoing Management

With your team-project data file loaded, you can:

Team Management

  • Add new members - Bring in new contributors and assign them to appropriate tiers
  • Edit member details - Update contact info, payment preferences, or contribution notes
  • Promote members - Move contributors to higher tiers based on increased impact (promotions are permanent!)
  • Remove members - Only in extreme cases of bad behavior (see Bad Actors policy)

Payout Calculations

  • Mock Payouts - Calculate hypothetical distributions to preview how money would be split
  • Real Payouts - Calculate actual distributions and commit them to the timeline
    • Enter the total revenue amount after platform cuts
    • Review the automated calculations showing each member's share
    • Verify everything looks correct
    • Commit the payout to create a permanent record in the timeline

Reports & Transparency

  • Export payout logs - Generate reports showing:
    • Individual payout amounts for the latest cycle
    • Total earnings per member across all payouts
    • Complete transaction history
  • Share with team - Export these logs to show team members how distributions were calculated

üí° Important

Always save a new copy of your team-project data file after making any changes. This ensures you don't lose your updates and creates a backup history.

5Processing Actual Payouts

When it's time to distribute money to your team:

  1. Calculate in the app - Run the payout calculation with your actual revenue amount
  2. Review the breakdown - Verify all amounts and tier assignments are correct
  3. Commit to timeline - Save this payout as a permanent record
  4. Process payments manually - Use your preferred payment method (bank transfer, PayPal, Stripe, etc.) to send each member their share
  5. Document everything - Take screenshots of transactions, save receipts, and update your transparency folder
  6. Export and share - Generate a payout log and share it with contributing members so they can verify the distribution

Remember: The web app calculates the splits, but you still manually send the payments. This gives you full control and ensures compliance with your local banking and tax requirements.

6Repeat & Maintain

From here, your workflow becomes a cycle:

  1. Load your latest team-project data file when opening the app
  2. Make any needed updates (add members, promotions, etc.)
  3. Calculate payouts when revenue reaches thresholds
  4. Process and document the distributions
  5. Save an updated team-project data file
  6. Share updated file and logs with team for transparency

This process continues for the lifetime of your project, ensuring fair and transparent revenue sharing forever.

Best Practices

File Management

  • Keep backups - Store copies of your team-project data files in multiple locations (external drive, cloud storage, etc.)
  • Use version control - The automatic date-time filenames help you track which is the latest version
  • Share read-only copies - Give contributing members access to the latest file so they can verify fairness

Transparency

  • Document all payouts - Take screenshots of platform earnings and payment confirmations
  • Maintain a shared folder - Use Google Drive or similar to give team view-only access to all records
  • Communicate regularly - Let team members know when payouts are coming and when they've been processed

Security & Privacy

  • Minimize personal info - Only include what's necessary (name/alias, email, role, tier)
  • Scrub sensitive data - Remove bank account numbers, full addresses, or other identifying information before sharing files
  • Vet new members - Be careful who gets access to team files and transparency folders

Troubleshooting

Can't load my data file?

Make sure you're using the correct file format (typically .json). If the file is corrupted, restore from a backup copy.

Calculations seem wrong?

Double-check your team composition and tier assignments. The self-adjusting algorithm may be activating if you have many assistant/special thanks members compared to main tier members.

Lost my data file?

This is why backups are crucial! Check your computer's recent files, cloud storage, or transparency folder for the latest copy. If all else fails, you'll need to recreate your data from documentation.
